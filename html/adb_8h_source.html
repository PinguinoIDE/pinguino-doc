1 //
2 // This file implements the ADB protocol (host side).
3 // It hides the underlying USB layer completely and exposes a simple API that
4 // enables the client to open several independant communication channels with
5 // the remote side. These communication channels can correspond to virtual TCP
6 // ports on the remote side, a command shell, etc. Each such communication
7 // channel is able to process a single buffer of outgoing data at a given moment
8 // as well as calls back a client-provided function as soon as incoming data
9 // arrives.
10 // The number of concurrently open channels is limited by ADB_MAX_CHANNELS.
11 //
12 // Closing of a channel:
13 // ---------------------
14 // In case the channel opening is rejected by the remote end or closed by it
15 // later, the client will be notified by calling the channel's callback function
16 // with a NULL data buffer and a length of 0.
17 // Client may close a channel by calling the ADBClose() function.
18 //
19 // Error handling:
20 // ---------------
21 // This module automatically attempts to establish a connection with the remote
22 // end as soon as USB layer is attached. When the connection drops, the
23 // connection will be dropped, and re-established when possible.
24 // The client should check the return code on every call to ADBTasks(). When the
25 // connection drops, all state is lost and any open channels are closed. They
26 // will be notified for the closure as mentioned in the above section.
27 // Upon detection of any sort of error or data corruption during connection,
28 // this module will reset the USB device and start over. Under normal operation
29 // this should never happen, so the client may want to reset everything and
30 // restart the program whenever a connection drops, but this is of course not
31 // required.
32 //
33 // Important note:
34 // ---------------
35 // The USB layer does not handle transmision of ROM buffers. Do not pass ROM
36 // buffers to ADBWrite(). When defining an array as const, the compiler
37 // typically stores it in ROM, so a possible workaround is to define it as non-
38 // const, even if it does not change.
39 //
40 // Usage example (error handling omitted):
41 // ---------------------------------------
42 // void ShellChannelCallback(ADB_CHANNEL_HANDLE h, void* data, UINT32 data_len) {
43 // print(data, data_len);
44 // }
45 //
46 // ...
47 // ADBInit(); // call only once.
48 // while (!ADBTasks());
49 // h = ADBOpen("shell:", &ShellChannelCallback);
50 // while (!ADBChannelReady(h)) {
51 // ADBTasks();
52 // }
53 // ADBWrite(h, data, sizeof data);

55 // ADBTasks();
56 // }
